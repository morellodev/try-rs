# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A native Rust port of [`tobi/try`](https://github.com/tobi/try) — an ephemeral workspace manager. The user types `try`, picks (or creates) a date-prefixed directory in the TUI selector, and the shell `cd`s into it. The binary itself can't change the parent shell's `cwd`, so it emits a shell script on stdout that the wrapper function (installed via `try init`) `eval`s.

The project is feature-complete against the upstream Ruby implementation: **387/387 upstream spec tests pass** plus 117 internal unit tests.

## Build, test, run

```sh
cargo build --release                                  # binary at target/release/try
cargo test                                             # 117 unit + doc tests
cargo test --lib shell::emit                           # one module's tests
cargo clippy --all-targets --release                   # lint
./spec/tests/runner.sh ./target/release/try            # full upstream spec runner (387 tests)

# Try the binary interactively in the current shell:
eval "$(./target/release/try init)"
try                                                    # opens the TUI
```

Snapshot tests in `shell::emit` can be regenerated when the wrapper contract intentionally changes:

```sh
SNAPSHOTS=overwrite cargo test --lib shell::emit
```

## Architecture: the shell-script contract

The whole design pivots on **stdout is a shell script, stderr is the UI**:

- `selector::run` + `dialog::*` produce an `action::Action` enum (`Cd`, `Mkdir`, `Clone`, `Worktree`, `Delete`, `Rename`, `Graduate`).
- `shell::emit::emit` serializes an `Action` to stdout as a `&& \`-chained POSIX script with a `# if you can read this…` warning header.
- The shell wrapper (generated by `shell::init::wrapper` per shell — bash/zsh/fish/pwsh) captures that stdout: exit 0 → `eval` it, non-zero → echo it (used for "cancelled" / errors).
- The TUI writes only to **`io::stderr()`**, wrapped via `anstream::AutoStream` (`always` normally, `never` for `--no-colors` / `NO_COLOR`). Never write UI to stdout — it'll break `eval`.

This split is the most important invariant in the codebase.

## Module layers (inward-flowing dependencies)

```
main.rs ─────────────────────── clap dispatch, env resolution
  │
  ├─ tui::{terminal, input, render, selector, dialog}    interactive layer
  │                                                       (no fs writes; produces Action)
  │
  ├─ discover ─── workspace, clock, naming               filesystem scan + scoring
  │
  ├─ shell::{posix, emit, init, kind}                    Action → script bytes
  │
  └─ action, fuzzy, git, naming, clock, workspace        pure cores (no I/O)
```

- **`tui::selector::apply_key`** is a pure state machine — unit-tested without a terminal. The I/O loop (`selector::run`) is a thin shell around it.
- **`tui::input::KeySource`** trait has two impls: `TerminalKeys` (production, crossterm) and `ScriptedKeys` (test injection via `--and-keys`). This is what lets the upstream spec runner drive the TUI non-interactively.
- **`fuzzy::Matcher`** implements the upstream scoring algorithm verbatim — char match + word-boundary + proximity bonuses, density × length multipliers; `Workspace.base_score` (date-prefix + recency, computed once at scan time) is added afterward.
- **`workspace::WorkspaceRoot`** is a newtype enforcing `is_absolute()`. Same for `ProjectsRoot`. Don't take raw `PathBuf` at the API surface.

## Test-injection flags (drive the TUI non-interactively)

All hidden from `--help`; used by the spec runner and ad-hoc debugging.

| Flag | What |
|---|---|
| `--and-type QUERY` | prefill the search buffer (overrides positional query) |
| `--and-keys SPEC` | scripted keystrokes; token grammar (`UP,DOWN,CTRL-D,TYPE=foo`) or raw bytes |
| `--and-exit` | render one frame, then exit |
| `--and-confirm TEXT` | short-circuit delete-confirm dialog with this string (`YES` = confirm) |

When any of these are set, `exec_cd` skips `terminal::Guard` entirely so captured stderr is unwrapped ANSI bytes (no alt-screen, no raw mode). See `parse_keys_spec` in `src/tui/input.rs` for the grammar.

## Spec compliance harness

`spec/` is vendored from upstream `tobi/try` and pinned. The runner script (`spec/tests/runner.sh`) is the ground truth — it exercises **byte-exact** shell-script output, render-byte counts, ANSI sequences, and dialog flows. Before claiming a non-trivial behavioral change is correct, run it.

Several upstream tests inspect specific byte patterns (e.g. `\e[1[m;]` to detect SGR — but also accidentally matches `\e[1;1H`). The `tui::render` module sidesteps these by emitting cursor home as literal `\x1b[H` rather than `MoveTo(0,0)`, and skipping the redundant `MoveTo(0,0)` after clear.

## Conventions worth knowing

- **No backgrounds on selector rows** for accessibility. Selected = bold arrow; marked = bold red foreground; the only colored background is the DELETE MODE footer (red bg + white bold fg). Emphasis comes from typography (bold), icon (📁/🗑️/🔗), and the arrow, never from row-bg/fg pairs.
- **POSIX quoting in `shell::posix::quote` must stay byte-identical to upstream** — the spec runner greps for exact single-quoted forms. Don't replace with `shell-escape` or similar.
- **`emit::emit` reads `current_dir()` for the `Delete` "cd back if pwd was removed" command.** For deterministic tests, use `emit::emit_with_pwd(out, action, Some(path))`. All `shell::emit::tests::*_snapshot` tests use the latter.
- **The `--path` global flag and `Init.explicit_path` positional are *different things*** despite both being paths: `--path` sets the runtime tries root (and the *default* baked into the wrapper for `init`), while `Init.explicit_path` is the *hard-coded* path embedded in the wrapper (skips `$TRY_PATH` at call time). Renaming the field to `explicit_path` was necessary because clap silently shared the value when both were named `path`.
- **Rust 1.91+ required** — `str::floor_char_boundary` / `ceil_char_boundary` (used in `tui::selector` and `tui::dialog` for UTF-8-safe line editing).
